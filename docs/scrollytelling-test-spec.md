# Scrollytelling Test Spec

## Goal

Add one small scrollytelling section to the portfolio without turning the site into a complicated presentation. The section should make the professional story clearer as the visitor scrolls.

## Story

The scrollytelling section should explain four moves:

1. Start with a confusing real-world task.
2. Turn the task into a clear product question.
3. Build a working web project with real tools.
4. Revise and publish proof that someone can review.

## Acceptance Criteria

- The section works inside the existing static site.
- It uses the current visual style and stays simple/professional.
- It has a sticky visual area on larger screens.
- It still reads normally on mobile.
- Space, ArrowDown, and PageDown advance the story when the section is visible.
- Mobile tapping inside the story section advances to the next slide.
- It does not replace the project cards, skills, or contact sections.
- It includes a small script only for highlighting the active scroll step.
- The README includes the hosted GitHub Pages link.
